1. Yönetici Özeti

Sistemin iki yüzü test edilmiştir:

  • Frontend (Client): İstemci tarafı güvenlik kontrollerinde tam bir çöküş (Bypass) tespit edilmiştir. Next.js mimarisi kandırıldı.
  • Backend (Supabase): Çekirdek veriler (Restoranlar vb.) başarıyla korunmuş, ancak ek tablolarda ölümcül güvenlik zafiyetleri görülmüştür.
  • XSS İstismarı: Korumasız veritabanı kısımlarından siteye doğrudan zararlı JavaScript enjeksiyonu gerçekleştirilmiştir.
KAYNAK KOD SIZINTISI

AUTH BYPASS

3. Tarayıcıya Güvenmek

Uygulamanın Admin Paneli, arka uçtan doğrulanmış bir mühür almak yerine Frontend'de JavaScript ile kimlik sormaktadır.

  • Sistemin Beklentisi: LocalStorage'dan Token alınır, fetch ile kontrol edilir.
  • Siber Saldırımız: Tarayıcıdaki fetch fonksiyonu "Man-in-the-Middle" yöntemiyle hacklendi!
  • Sonuç: Sunucu reddetse de vizyonel olarak Admin Paneli %100 açıldı.

BAŞARILI SAVUNMA

7. İyi Haber: RLS Duvarı

Admin paneli kırılsa bile, geliştirici ekibi harika bir arka uç mimarisi (Supabase RLS) tasarlamış!

  • restaurants, blog_posts, admins gibi tablolar SADECE backend JWT imzası ile işlem kabul ediyor.
  • Sisteme yolladığımız tüm zararlı kayıt-silme denemeleri 42501 Unauthorized hatasıyla reddedildi.
  • Bu kural, sistemin göçmesini engelledi.
XSS DEFACEMENT

8. Facia: Unutulan Tablolar

Ağır zırhlara rağmen, küçük tablolarda RLS kalkanı unutulmuş!

  • Açık Tablolar: tags ve restaurant_tags
  • İstismar: İçeriye izinsiz olarak <script>alert('XSS')</script> kodu başarıyla enjekte edilmiştir.
  • Sitedeki Farvale adlı restoranın verisine bu virüs etiketi eklenmiş ve veri kalıcı olarak değiştirilmiştir. (Sonradan HACKED BY AUDIT TEST olarak revize edildi)
API SIZINTISI

9. Ölümcül Hata: Önbellek

Hacklenen bir eylemin Next.js sistemine yansıması normalde zaman alırdı, ancak...

  • /api/revalidate endpointi şifresiz ve herkese açık!
  • Saldırganlar bu API'yi çağırarak sitenin tüm önbelleğini sıfırlayıp, yazdıkları virüslü (XSS) etiketleri anında canlıya geçirdiler.
  • Bu açık DDoS ve Vercel fatura şişirme saldırılarına yol açar.
DATA LEAK

10. Sessiz Sızıntılar

Sistemde admin yetkisi (veya herhangi bir JWT) olmadan anonim şekilde çekilebilen şok edici veriler bulunmuştur:

  • Profiller İfşa Olmuş: profiles tablosu herkese açık (RLS kapalı). Tüm üyelerin (örn: 60aydoganosman) Kullanıcı Adı, UUID'leri ve kayıt tarihleri çekilebiliyor.
  • Aşırı SSR Veri Dökümü: Next.js Server-Side-Rendering esnasında, 323 restoranın ve tüm blogların arka plan verilerini sayfa HTML kodu içerisine (self.__next_f.push(...) ile) gömüyor. Herkes botlarla saniyeler içinde tüm veritabanını kopyalayabilir.
  • RPC Korumasızlığı: Veritabanınızın iç istatistik fonksiyonları (Örn: /rpc/top_tags) anonim POST çağrılarına tamamen açıktır.

11. Kesin Çözüm Reçeteleri

  • Server-Side Admin Kontrolü: Client'a güvenmeyin. middleware.ts içerisinde supabase.auth.getSession() kullanın.
  • Güvenlik Kalkanı: Supabase'den tags ve restaurant_tags tabloları için "Enable RLS" kuralını devreye alın.
  • Gizli API: /api/revalidate dosyasına ?secret=XXX yetki koşulu ekleyin.
